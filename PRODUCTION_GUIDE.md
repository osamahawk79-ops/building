# دليل نشر منصة التعليم الإلكتروني على الإنترنت (Production Deployment Guide)

لتشغيل المنصة على الإنترنت وإتاحتها للطلاب والمستخدمين، يرجى اتباع هذا الدليل المخصص لتجهيز وإطلاق نسخة الإنتاج.

---

## 1. الملفات المطلوبة للرفع إلى السيرفر (Production Files)

عند رفع المنصة إلى خادم الإنترنت (مثل VPS أو Cloud Server)، لا تحتاج لرفع ملفات التطوير بالكامل. الملفات والمجلدات المطلوبة للتشغيل هي:

```text
📁 dist/               # المجلد الناتج بعد عملية البناء (يحتوي على كود الفرونتيد والباكيند المترجم)
📁 data/               # يحتوي على قاعدة البيانات db.json وإعدادات الموقع
📁 storage/            # يحتوي على المرفقات، الصور، الفيديوهات المصغرة، والملفات المرفوعة
📄 package.json        # ملف التعريف لتشغيل السيرفر وحزم العمل
📄 .env                # ملف متغيرات البيئة السرية الخاص بالسيرفر
```

> [!IMPORTANT]
> لا تقم برفع مجلد `node_modules` أو ملفات التطوير مثل `src/` أو `server/` أو `electron/` إلى خادم الإنتاج لضمان السرعة والأمان.

---

## 2. إعداد ملف البيئة للإنتاج (.env)

قم بإنشاء ملف `.env` على السيرفر وضبط المتغيرات التالية:

```env
# منفذ تشغيل المنصة (الافتراضي 3000)
PORT=3000

# وضع التشغيل (يجب أن يكون production)
NODE_ENV=production

# المفتاح السري لتشفير الجلسات و tokens (اكتب مفتاحاً عشوائياً قوياً)
JWT_SECRET=super_secret_key_change_me_in_production

# إعدادات تسجيل الدخول بجوجل (اختياري)
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret
GOOGLE_REDIRECT_URI=https://your-domain.com/api/auth/google/callback
```

---

## 3. طريقة بناء وتشغيل المنصة على السيرفر (Linux/VPS)

خوادم الويب تعمل عادة بنظام **Linux** وإصدارات Node.js مستقرة (LTS)، لذا فإن عملية البناء ستتم بسلاسة تامة. اتبع الخطوات التالية في سطر الأوامر بالسيرفر:

### الخطوة 1: تثبيت الحزم المطلوبة للإنتاج فقط
```bash
npm install --omit=dev
```

### الخطوة 2: بناء المشروع
إذا كنت ترفع الكود المصدري كاملاً وتريد البناء على السيرفر مباشرة:
```bash
npm run build
```

### الخطوة 3: تشغيل السيرفر في الخلفية
لتشغيل السيرفر وضمان بقائه يعمل حتى بعد إغلاق Terminal، يفضّل استخدام مدير العمليات **PM2**:

1. تثبيت PM2:
   ```bash
   npm install -g pm2
   ```
2. تشغيل التطبيق باسم مخصص (مثال: `lms-platform`):
   ```bash
   pm2 start dist/server.cjs --name "lms-platform"
   ```
3. لضمان تشغيل السيرفر تلقائياً عند إعادة تشغيل جهاز السيرفر:
   ```bash
   pm2 startup
   pm2 save
   ```

## ⚠️ ملاحظة بخصوص البناء المحلي (Vite Build Local Error)
إذا حاولت تشغيل أمر البناء محلياً (`npm run build`) على جهاز Windows الخاص بك وظهر خطأ **Heap Corruption (EXIT: -1073740791)**:
- **السبب:** هذا الخطأ ناتج عن استخدام نسخة Node.js تجريبية/غير مستقرة مثبتة حالياً على جهازك وهي **Node v24.18.0**. حيث تتعارض هذه النسخة مع مكتبات التجميع الأصلية (Native Bundle Libraries) الخاصة بـ Rollup على Windows.
- **الحل:** لن تواجه هذه المشكلة على سيرفر الإنتاج (الذي يعمل بنسخة مستقرة ونظام Linux). ولكن إذا أردت البناء محلياً على جهازك بنجاح، يرجى فقط تثبيت نسخة Node.js مستقرة (LTS) مثل **Node v20.x** أو **Node v22.x** بدلاً من الإصدار v24.
