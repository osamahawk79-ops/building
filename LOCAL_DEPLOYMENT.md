# دليل تشغيل المنصة كخادم محلي (Local On-Premise Server)

تم تهيئة المشروع للعمل بالكامل من جهازك الشخصي، مما يغني عن استخدام الخدمات السحابية الخارجية. إليك التعليمات البرمجية والخطوات اللازمة للتشغيل الدائم والتأمين.

## 1. متطلبات التشغيل
- تثبيت **Node.js** (إصدار 18 فما فوق).
- تثبيت **npm** (يأتي مع Node).
- تثبيت **PM2** للتشغيل الدائم في الخلفية:
  ```bash
  npm install pm2 -g
  ```

## 2. إعداد السيرفر المحلي
تم تعديل الكود ليدعم:
- **CORS**: يسمح بالاتصال من أي مصدر محلي.
- **Security**: تم دمج `Helmet` لحماية الرؤوس (Headers) و `Rate Limiting` لمنع هجمات الإغراق.
- **Storage**: يتم حفظ كافة الفيديوهات والملفات في مجلد `storage` بداخل مجلد المشروع مباشرة.

## 3. التشغيل الدائم باستخدام PM2
لضمان بقاء السيرفر يعمل حتى بعد إغلاق نافذة الـ Terminal:
1. قم ببناء المشروع أولاً:
   ```bash
   npm run build
   ```
2. قم بتشغيل السيرفر باستخدام PM2:
   ```bash
   pm2 start dist/server.cjs --name "maharat-lms"
   ```
3. لحفظ القائمة لتعمل تلقائياً عند إعادة تشغيل الجهاز:
   ```bash
   pm2 save
   pm2 startup
   ```

## 4. ربط الدومين وتوجيه المنافذ (Port Forwarding)
لكي يتمكن المستخدمون من تصفح الموقع من خارج شبكتك المحلية:
1. **Static IP**: تأكد من أن جهازك يمتلك IP ثابت بداخل الشبكة (من إعدادات الراوتر).
2. **Port Forwarding**: ادخل إلى إعدادات الراوتر وقم بتوجيه المنفذ **3000** (أو المنفذ الذي اخترته) إلى الـ IP المحلي لجهازك.
3. **Domain Name**: قم بربط الدومين الخاص بك بالـ **Public IP** الخاص بجهازك (يمكنك معرفته عبر موقع `whatsmyip.org`).
   - إذا كان الـ IP يتغير، استخدم خدمة **Dynamic DNS** مثل No-IP.

## 5. تأمين الجهاز
- تم تفعيل **Helmet** و **Express Rate Limit** برمجياً.
- ننصح بوضع السيرفر خلف **Nginx** كـ Reverse Proxy إذا كنت تتوقع عدداً كبيراً من المستخدمين، لإضافة طبقة SSL وتأمين إضافي.

## 6. مسارات الملفات
- **البيانات**: `data/db.json`
- **الوسائط**: `storage/videos`, `storage/attachments`, `storage/branding`
